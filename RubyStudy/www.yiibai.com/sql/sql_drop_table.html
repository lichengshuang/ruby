SQL丢弃或删除表

SQL丢弃或删除表实例代码教程 - 使用SQL DROP TABLE语句删除表的定义和所有数据，索引，触发器，约束和该表的权限规范。

使用SQL DROP TABLE语句删除表的定义和所有数据，索引，触发器，约束和该表的权限规范。

注: 使用此命令时要小心，因为一旦一个表被删除表中提供的所有信息也将被永远失去了。

语法:

DROP TABLE语句的基本语法如下： 

DROP TABLE table_name;

例子:

让我们先来验证CUSTOMERS表，然后我们将它从数据库中删除： 

SQL> DESC CUSTOMERS;
+---------+---------------+------+-----+---------+-------+
| Field   | Type          | Null | Key | Default | Extra |
+---------+---------------+------+-----+---------+-------+
| ID      | int(11)       | NO   | PRI |         |       |
| NAME    | varchar(20)   | NO   |     |         |       |
| AGE     | int(11)       | NO   |     |         |       |
| ADDRESS | char(25)      | YES  |     | NULL    |       |
| SALARY  | decimal(18,2) | YES  |     | NULL    |       |
+---------+---------------+------+-----+---------+-------+
5 rows in set (0.00 sec)

这意味着CUSTOMERS表是在数据库中，让我们把它如下： 

SQL> DROP TABLE CUSTOMERS;
Query OK, 0 rows affected (0.01 sec)

现在如果你想尝试DESC命令，那么你会得到如下的错误： 

SQL> DESC CUSTOMERS;
ERROR 1146 (42S02): Table 'TEST.CUSTOMERS' doesn't exist

这里的测试是我们在例子中使用的数据库的名称。
