JDBC Statements

JDBC Statement实例代码教程 - 一旦连接，我们可以与数据库进行交互。 JDBC Statement，CallableStatement对象和PreparedStatement接口中定义的方法和属性能使你能够从数据库SQL或PL/SQL命令发送和接收数据。

一旦连接，我们可以与数据库进行交互。JDBC Statement，CallableStatement对象和PreparedStatement接口中定义的方法和属性能使你能够从数据库SQL或PL/SQL命令发送和接收数据。

他们还定义的方法，帮助桥接的数据类型在数据库中使用Java和SQL数据类型之间的差异。

下表总结了每个接口的目的，了解你是怎么决定要使用的接口吗？

接口 推荐使用
Statement 使用通用访问你的数据库。当你使用静态SQL语句在运行时。 Statement接口不能接受的参数。
PreparedStatement 当你打算使用的SQL语句多次。在运行时，PreparedStatement接口接受输入参数。
CallableStatement 使用时，您要访问的数据库的存储过程。 CallableStatement接口还可以接受运行时的输入参数。

Statement对象:

创建Statement 对象:

在可以使用Statement对象执行SQL语句，需要创建一个使用Connection对象的createStatement()方法，如下面的例子： 

Statement stmt = null;
try {
   stmt = conn.createStatement( );
   . . .
}
catch (SQLException e) {
   . . .
}
finally {
   . . .
}

一旦创建了一个Statement对象，就可以用它来执行一个SQL语句，它的三个执行方法之一。

  1. boolean execute(String SQL) : 返回一个布尔值true，如果一个ResultSet对象可以被检索;否则，返回false。使用此方法来执行SQL DDL语句，或当你需要使用真正的动态SQL。

  2. int executeUpdate(String SQL) : 返回执行的SQL语句所影响的行的数目。使用此方法执行SQL语句，希望得到一个影响的行数 - 例如，INSERT，UPDATE或DELETE语句。

  3. ResultSet executeQuery(String SQL) : 返回的ResultSet对象。使用这种方法时，你希望得到一个结果集，你会用一个SELECT语句。

关闭Statement 对象:

正如关闭一个Connection对象，以节省数据库资源，出于同样的原因，也应该关闭Statement对象。

一个简单的调用close()方法将完成这项工作。如果您关闭了Connection对象，首先它会关闭Statement对象。然而，你应该始终明确地关闭Statement对象，以确保适当的清理。 

Statement stmt = null;
try {
   stmt = conn.createStatement( );
   . . .
}
catch (SQLException e) {
   . . .
}
finally {
   stmt.close();
}

PreparedStatement 对象:

PreparedStatement接口扩展Statement接口，让附加功能的优势通过一个通用Statement对象。

此语句也给你提供的参数动态的灵活性。

创建PreparedStatement 对象: 

PreparedStatement pstmt = null;
try {
   String SQL = "Update Employees SET age = ? WHERE id = ?";
   pstmt = conn.prepareStatement(SQL);
   . . .
}
catch (SQLException e) {
   . . .
}
finally {
   . . .
}

JDBC中的所有参数表示的吗？符号作为参数标记，这是大家知道的。在执行SQL语句之前，必须提供每个参数的值。

setXXX()方法的参数，其中XXX表示你希望绑定的输入参数的值的Java数据类型的值绑定到。如果忘记了提供的值，将收到一个SQLException。

每个参数标记是指由序数位置。第一标记表示位置1时，下一个位置2，等等。这种方法不同于Java的数组的索引，从0开始。
所有Statement对象与数据库进行交互（a）execute()，（b）executeQuery() （c）PreparedStatement对象的executeUpdate()方法。然而，这些方法修改为使用SQL语句，可以输入参数。

关闭PreparedStatement 对象:

正如你关闭Statement对象，出于同样的原因，你也应该关闭的PreparedStatement对象。

一个简单的调用close()方法将完成这项工作。如果关闭了Connection对象，首先它会关闭PreparedStatement对象为好。然而，应该始终明确关闭PreparedStatement对象，以确保适当的清理。 

PreparedStatement pstmt = null;
try {
   String SQL = "Update Employees SET age = ? WHERE id = ?";
   pstmt = conn.prepareStatement(SQL);
   . . .
}
catch (SQLException e) {
   . . .
}
finally {
   pstmt.close();
}

CallableStatement 对象:

正如一个Connection对象创建Statement和PreparedStatement对象时，它也将被用来执行调用数据库存储过程创建CallableStatement对象。

创建CallableStatement 对象:

假设，需要执行以下Oracle存储过程： 

CREATE OR REPLACE PROCEDURE getEmpName 
   (EMP_ID IN NUMBER, EMP_FIRST OUT VARCHAR) AS
BEGIN
   SELECT first INTO EMP_FIRST
   FROM Employees
   WHERE ID = EMP_ID;
END;

注: 上面的存储过程已经被甲骨文写的，但我们正在使用MySQL数据库，我们写相同的存储过程用于MySQL如下EMP数据库中创建它： 

DELIMITER $$

DROP PROCEDURE IF EXISTS `EMP`.`getEmpName` $$
CREATE PROCEDURE `EMP`.`getEmpName` 
   (IN EMP_ID INT, OUT EMP_FIRST VARCHAR(255))
BEGIN
   SELECT first INTO EMP_FIRST
   FROM Employees
   WHERE ID = EMP_ID;
END $$

DELIMITER ;

存在三种类型的参数：IN，OUT，INOUT。只使用PreparedStatement对象的IN参数。 CallableStatement对象可以使用所有三个。
每个这里的定义是：

Parameter Description
IN A parameter whose value is unknown when the SQL statement is created. You bind values to IN parameters with the setXXX() methods.
OUT A parameter whose value is supplied by the SQL statement it returns. You retrieve values from theOUT parameters with the getXXX() methods.
INOUT A parameter that provides both input and output values. You bind variables with the setXXX() methods and retrieve values with the getXXX() methods.

下面的代码片段显示了如何利用的Connection.prepareCall()方法实例化一个CallableStatement对象，根据上述存储过程： 

CallableStatement cstmt = null;
try {
   String SQL = "{call getEmpName (?, ?)}";
   cstmt = conn.prepareCall (SQL);
   . . .
}
catch (SQLException e) {
   . . .
}
finally {
   . . .
}

String变量SQL的存储过程，参数占位符。

使用CallableStatement对象很像使用PreparedStatement对象。必须执行该语句之前的所有参数值绑定，否则将收到一个SQLException。

如果有IN参数，只要遵循相同的规则和技术，适用于PreparedStatement对象中，使用的setXXX()方法，对应于要绑定的Java数据类型。

当使用OUT和INOUT参数，必须采用额外的CallableStatement对象的registerOutParameter()方法，registerOutParameter()方法绑定的数据类型的存储过程返回JDBC数据类型。

一旦你调用存储过程，检索OUT参数值与相应的getXXX()方法。这种方法检索到的值的SQL类型转换到Java的数据类型。

关闭CallableStatement 对象:

正如关闭其他Statement对象，出于同样的原因，也应该关闭CallableStatement对象。

一个简单的调用close()方法将完成这项工作。如果关闭了Connection对象，首先它会关闭CallableStatement对象为好。然而，应该始终明确地关闭CallableStatement对象，以确保适当的清理。 

CallableStatement cstmt = null;
try {
   String SQL = "{call getEmpName (?, ?)}";
   cstmt = conn.prepareCall (SQL);
   . . .
}
catch (SQLException e) {
   . . .
}
finally {
   cstmt.close();
}
