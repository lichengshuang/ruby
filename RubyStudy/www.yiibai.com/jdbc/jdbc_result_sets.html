JDBC ResultSet结果集

JDBC ResultSet结果集实例代码教程 - ResultSet对象维护一个游标指向结果集的当前行。 “结果集”术语是指ResultSet对象中包含的数据的行和列。

读取数据从数据库中查询的SQL语句返回的数据结果集。SELECT语句从数据库中选择行和查看他们的结果集是标准的方式。 java.sql.ResultSet接口的数据库查询的结果集。
ResultSet对象维护一个游标指向结果集的当前行。 “结果集”术语是指ResultSet对象中包含的数据的行和列。

ResultSet接口中方法可以被分解为三个类别：

  1. 导航方法: 用于移动光标。

  2. Get方法: 用于查看由光标指向的当前行的列中的数据。

  3. Update方法: 用于更新的当前行的列中的数据。更新可以在基础数据库中更新。

光标移动的ResultSet的属性的基础上。这些属性在创建时指定相应的语句生成的ResultSet。

JDBC提供了以下连接方法来创建报表所需的ResultSet：

  1. createStatement(int RSType, int RSConcurrency);

  2. prepareStatement(String SQL, int RSType, int RSConcurrency);

  3. prepareCall(String sql, int RSType, int RSConcurrency);

第一个参数表示ResultSet对象的类型，第二个参数是用于指定是否是只读或可更新的结果集ResultSet常量之一。

ResultSet类型:

以下是可能RS类型，如果你不指定任何ResultSet类型，将自动获得一个TYPE_FORWARD_ONLY。

Type Description
ResultSet.TYPE_FORWARD_ONLY 光标只能向前的结果集。
ResultSet.TYPE_SCROLL_INSENSITIVE 游标可以向前和向后滚动，与他人发生后的结果集的数据库所做的更改是不敏感的结果集。
ResultSet.TYPE_SCROLL_SENSITIVE. 游标可以向前和向后滚动，并且结果集由他人发生后的结果集的数据库的变化是敏感的。

并发ResultSet:

可能RS是并发的，如果你不指定任何并发类型，您将自动获得一个CONCUR_READ_ONLY。

Concurrency 描述
ResultSet.CONCUR_READ_ONLY 创建一个只读的结果集。这是默认的
ResultSet.CONCUR_UPDATABLE 创建一个可更新的结果集。

我们所有的例子中可以写成书面至今的如下初始化一个Statement对象创建仅向前，只读的ResultSet对象： 

try {
   Statement stmt = conn.createStatement(
                           ResultSet.TYPE_FORWARD_ONLY,
                           ResultSet.CONCUR_READ_ONLY);
}
catch(Exception ex) {
   ....
}
finally {
   ....
}

S.N. Methods & Description

将光标移动到刚刚结束的最后一排

移动光标返回到当前行，如果光标是目前在插入行，否则，该方法不执行任何操作

ResultSet接口中含有几十种方法获取当前行的数据。

有一个get方法为每个可能的数据类型，每个get方法有两个版本：

  1. 一个带列名的形式。

  2. 一个带列索引的形式。

例如，如果你得到的列包含一个int，可以使用调用getInt()方法的结果：

S.N. Methods & Description
1 public int getInt(String columnName) throws SQLException
返回int在当前行中指定列的ColumnName
2 public int getInt(int columnIndex) throws SQLException
返回当前行中指定列索引的int。列索引从1开始，这意味着为1的行的第一列的某一行，第二列是2，等等。

同样有获取ResultSet接口中的8个Java的基本数据类型的方法，以及常见的类型，例如java.lang.String，java.lang.Object，java.net.URL

还有一些方法获取SQL数据类型的java.sql.Date，将java.sql.Time的java.sql.Timestamp，java.sql.Clob，和java.sql.Blob的。使用这些SQL数据类型的更多信息，请查看文档。

更新结果集：

ResultSet接口包含一个更新的结果集的数据更新方法的集合。

与get方法一样，有两个更新为每个数据类型的方法：

  1. 一个带列名的形式。

  2. 一个带列索引的形式。

例如，要更新一个字符串列的结果集的当前行，你会使用以下updateString()方法之一：

S.N. Methods & Description
1 public void updateString(int columnIndex, String s) throws SQLException
Changes the String in the specified column to the value of s.
2 public void updateString(String columnName, String s) throws SQLException
Similar to the previous method, except that the column is specified by its name instead of its index.

有原始数据类型和String对象，URL，java.sql包中的SQL数据类型的更新方法。

更新的结果集行更改的ResultSet对象的当前行中的列，而不是在底层的数据库。要更新到数据库中的行，需要调用下面的方法。

S.N. 方法& 描述
1 public void updateRow()
更新，更新相应的数据库中的行的当前行。
2 public void deleteRow()
从数据库中删除当前行
3 public void refreshRow()
刷新的结果集，以反映最近的更改数据库中的数据。
4 public void cancelRowUpdates()
取消对当前行所做的任何更新。
5 public void insertRow()
数据库中插入一行。此方法只能被调用，当光标指向插入行。
