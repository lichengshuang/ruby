JDBC连接数据库

JDBC连接数据库实例代码教程 - 使用标准的JDBC包，它允许执行select，insert，update和delete的SQL语句，更新表中的数据。

在您安装相应的驱动程序，是时候建立一个数据库连接和使用JDBC。
建立一个JDBC连接的编程是相当简单的。下面这些简单的四个步骤：

  1. 导入JDBC包: import语句添加到Java程序在Java代码中导入所需的类。

  2. 注册JDBC 驱动: 这一步使JVM加载所需的驱动程序执行到内存中以便它能够满足JDBC请求。

  3. 数据库URL制定: 这是为了创建格式正确的地址，指向你要连接到的数据库。

  4. 创建连接对象: 最后，代码调用的DriverManager对象的getConnection()方法建立实际的数据库连接。

导入JDBC包:

import语句告诉Java编译器在哪里可以找到类，在代码中，一开始就被放置在源代码的头部。

要使用标准的JDBC包，它允许执行select，insert，update和delete的SQL语句，更新表中的数据，添加以下导入到你的源代码： 

import java.sql.* ;  // for standard JDBC programs
import java.math.* ; // for BigDecimal and BigInteger support

注册JDBC 驱动:

在程序中使用它之前，必须先注册驱动程序。注册驱动程序的过程如，Oracle驱动程序的类文件被加载到内存中以便它可以被用作JDBC接口的实现。

需要在程序中只有一次注册。可以注册一个驱动程序选用以下两种方法之一。

方法(I) - Class.forName():

注册驱动程序的最常用的方法是使用Java的Class.forName()方法来动态加载驱动程序的类文件到内存中，它会自动将其注册。这种方法是可取的，因为它可让您配置和便携的驱动程序注册。

下面的示例使用Class.forName()注册Oracle驱动程序： 

try {
   Class.forName("oracle.jdbc.driver.OracleDriver");
}
catch(ClassNotFoundException ex) {
   System.out.println("Error: unable to load driver class!");
   System.exit(1);
}

你可以使用getInstance()方法，以解决不符合要求的JVM，那么将有两个额外的例外情况如下代码： 

try {
   Class.forName("oracle.jdbc.driver.OracleDriver").newInstance();
}
catch(ClassNotFoundException ex) {
   System.out.println("Error: unable to load driver class!");
   System.exit(1);
catch(IllegalAccessException ex) {
   System.out.println("Error: access problem while loading!");
   System.exit(2);
catch(InstantiationException ex) {
   System.out.println("Error: unable to instantiate driver!");
   System.exit(3);
}

方法(II) - DriverManager.registerDriver():

你可以用它来注册一个驱动程序的第二种方法，使用静态DriverManager.registerDriver()方法。

如果使用的是一个不兼容的JDK JVM，如一个由Microsoft提供的应该使用registerDriver()方法。

以下的的示例使用registerDriver()注册Oracle驱动程序： 

try {
   Driver myDriver = new oracle.jdbc.driver.OracleDriver();
   DriverManager.registerDriver( myDriver );
}
catch(ClassNotFoundException ex) {
   System.out.println("Error: unable to load driver class!");
   System.exit(1);
}

数据库URL制定：

已经加载了驱动程序后，就可以使用DriverManager.getConnection()方法建立一个连接。为了便于参考，让列举的三个重载DriverManager.getConnection（）方法：

  1. getConnection(String url)

  2. getConnection(String url, Properties prop)

  3. getConnection(String url, String user, String password)

这里每一个表格都需要一个数据库URL。数据库的URL是指向数据库的地址。

制定一个数据库URL建立连接时出现的问题。

下表列出了比较常的JDBC驱动程序的名称和数据库URL。

RDBMS JDBC驱动名称 URL 格式
MySQL com.mysql.jdbc.Driver jdbc:mysql://hostname/ databaseName
ORACLE oracle.jdbc.driver.OracleDriver jdbc:oracle:thin:@hostname:port Number:databaseName
DB2 COM.ibm.db2.jdbc.net.DB2Driver jdbc:db2:hostname:port Number/databaseName
Sybase com.sybase.jdbc.SybDriver jdbc:sybase:Tds:hostname: port Number/databaseName

突出显示的部分是静态的URL格式为数据库设置，你需要更改仅剩余部分。

创建连接对象：

使用数据库URL，用户名和密码：

这里列出了三种形式的DriverManager.getConnection()方法来创建一个连接对象。最常用getConnection()的形式要求您传递一个数据库URL，用户名和密码，

假设你正在使用Oracle Thin驱动程序，您将指定的数据库的URL部分host:port:databaseName的值。

如果你有一台主机的TCP/IP地址192.0.0.1的主机名amrood，Oracle监听器配置为侦听端口1521上，数据库名称是EMP，然后完整的数据库URL将是： 

jdbc:oracle:thin:@amrood:1521:EMP

现在，你必须调用getConnection()方法用正确的用户名和密码，以获得一个Connection对象，如下： 

String URL = "jdbc:oracle:thin:@amrood:1521:EMP";
String USER = "username";
String PASS = "password"
Connection conn = DriverManager.getConnection(URL, USER, PASS);

只使用一个数据库URL：

DriverManager.getConnection()方法的第二种形式只需要一个数据库URL： 

DriverManager.getConnection(String url);

然而，在这种情况下，数据库URL包括用户名和密码，并具有下面的一般形式： 

jdbc:oracle:driver:username/password@database

因此，上面的连接，可以创建如下： 

String URL = "jdbc:oracle:thin:username/password@amrood:1521:EMP";
Connection conn = DriverManager.getConnection(URL);

使用数据库URL和Properties对象：

第三种形式的DriverManager.getConnection()方法需要一个数据库URL和一个Properties对象： 

DriverManager.getConnection(String url, Properties info);

一个Properties对象，保存了一组关键字值对。用来传递驱动程序属性中的驱动程序在调用getConnection()方法过程中。

为了使相同的连接由前面的例子中，使用下面的代码： 

import java.util.*;

String URL = "jdbc:oracle:thin:@amrood:1521:EMP";
Properties info = new Properties( );
info.put( "user", "username" );
info.put( "password", "password" );

Connection conn = DriverManager.getConnection(URL, info);

关闭JDBC链接:

在JDBC程序，每个数据库会话结束需要显式地关闭所有连接到数据库。但是如果忘记，Java的垃圾收集器将关闭连接时，它也会清除过时的对象。

依托垃圾收集，特别是在数据库编程，是非常不好的编程实践。你应该总是关闭与连接对象close()方法连接，这是一个好习惯。

为了保证连接被关闭，可以在你的代码中提供finally块。finally块总是会执行，无论是否发生异常或不。
关闭上面打开的连接，应该调用close()方法，如下所示： 


conn.close();

显式关闭的连接，节省DBMS资源。
