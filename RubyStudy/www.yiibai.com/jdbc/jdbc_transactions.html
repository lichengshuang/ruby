JDBC事务

JDBC事务实例代码教程 - 如果JDBC连接是在自动提交模式下，它在默认情况下，那么每一个SQL语句完成后提交到数据库。

如果JDBC连接是在自动提交模式下，它在默认情况下，那么每一个SQL语句完成后提交到数据库。
对于简单的应用程序上可能很好，但有三个原因，可能要关闭自动提交和管理事务：

  1. 为了提高性能

  2. 为了保持完整的业务流程

  3. 使用分布式事务

事务是否可以控制，以及何时更改应用到数据库中。它把一个单一的SQL语句或一组SQL语句作为一个逻辑单元，如果任何语句失败，整个事务失败。

要启用手动交易的JDBC驱动程序支持，而不是在自动提交模式下，默认情况下使用的，使用Connection对象的setAutoCommit()方法。如果你传递一个布尔值false的setAutoCommit()，关闭自动提交。可以通过一个布尔值，真正把它重新打开。

例如，如果你有一个名为conn的连接对象，代码如下，以关闭自动提交： 

conn.setAutoCommit(false);

提交&回滚

一旦已经完成了你的更改，需要提交更改，那么调用commit()方法连接对象如下： 

conn.commit( );

否则，回滚更新使用名为conn的连接到数据库，使用下面的代码： 

conn.rollback();

下面的例子演示了如何使用的提交和回滚对象： 

try{
   //Assume a valid connection object conn
   conn.setAutoCommit(false);
   Statement stmt = conn.createStatement();
   
   String SQL = "INSERT INTO Employees  " +
                "VALUES (106, 20, 'Rita', 'Tez')";
   stmt.executeUpdate(SQL);  
   //Submit a malformed SQL statement that breaks
   String SQL = "INSERTED IN Employees  " +
                "VALUES (107, 22, 'Sita', 'Singh')";
   stmt.executeUpdate(SQL);
   // If there is no error.
   conn.commit();
}catch(SQLException se){
   // If there is any error.
   conn.rollback();
}

在这种情况下，上述INSERT语句没有成功，一切都将被回滚。

使用保存点：

新的JDBC 3.0 Savepoint接口提供额外的事务控制。最现代的DBMS支持保存点在他们的环境中，如Oracle的PL/SQL。
当定义了一个逻辑的回退点，在一个事务中设置一个保存点。如果发生错误，可以回到过去的一个保存点，可以使用回滚方法撤消所有更改或保存点前所做的更改。


Connection对象有两个新的方法，可以帮助您管理保存点：

  1. setSavepoint(String savepointName): 定义了一个新的保存点。它也返回一个Savepoint对象。

  2. releaseSavepoint(Savepoint savepointName): 删除一个保存点。请注意，它需要一个Savepoint对象作为参数。这个对象通常是由setSavepoint()方法生成一个保存点。

有一个rollback ( String savepointName )方法可以回滚到指定的保存点。

下面的例子演示了如何使用一个Savepoint对象： 

try{
   //Assume a valid connection object conn
   conn.setAutoCommit(false);
   Statement stmt = conn.createStatement();
   
   //set a Savepoint - by http://www.yiibai.com/jdbc
   Savepoint savepoint1 = conn.setSavepoint("Savepoint1");
   String SQL = "INSERT INTO Employees " +
                "VALUES (106, 20, 'Rita', 'Tez')";
   stmt.executeUpdate(SQL);  
   //Submit a malformed SQL statement that breaks
   String SQL = "INSERTED IN Employees " +
                "VALUES (107, 22, 'Sita', 'Tez')";
   stmt.executeUpdate(SQL);
   // If there is no error, commit the changes.
   conn.commit();

}catch(SQLException se){
   // If there is any error.
   conn.rollback(savepoint1);
}

在这种情况下，上述INSERT语句没有成功，一切都将被回滚。
