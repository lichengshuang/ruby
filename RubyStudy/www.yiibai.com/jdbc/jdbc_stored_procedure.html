JDBC存储过程

JDBC存储过程实例代码教程 - 当一个Connection对象创建Statement和PreparedStatement对象时，它也将被用来执行调用数据库存储过程创建CallableStatement对象。

我们已经解释了如何使用存储过程，在JDBC讨论JDBC - Statements。本教程是该节相似，但它会给更多的信息有关JDBC的SQL转义语法。

当一个Connection对象创建Statement和PreparedStatement对象时，它也将被用来执行调用数据库存储过程创建CallableStatement对象。

创建CallableStatement 对象:

假设，需要执行以下Oracle存储过程： 

CREATE OR REPLACE PROCEDURE getEmpName 
   (EMP_ID IN NUMBER, EMP_FIRST OUT VARCHAR) AS
BEGIN
   SELECT first INTO EMP_FIRST
   FROM Employees
   WHERE ID = EMP_ID;
END;

注: 上面的存储过程是为Oracle写的，但我们正在使用MySQL数据库，以便让我们写相同的存储过程，在MySQL EMP数据库中创建它如下： 

DELIMITER $$

DROP PROCEDURE IF EXISTS `EMP`.`getEmpName` $$
CREATE PROCEDURE `EMP`.`getEmpName` 
   (IN EMP_ID INT, OUT EMP_FIRST VARCHAR(255))
BEGIN
   SELECT first INTO EMP_FIRST
   FROM Employees
   WHERE ID = EMP_ID;
END $$

DELIMITER ;

存在三种类型的参数：IN，OUT，INOUT。只使用PreparedStatement对象的IN参数。 CallableStatement对象可以使用上面提到的三个。

它们每个这里分别定义是：

参数 描述
IN 创建SQL语句的参数，其值是未知的。可以将值绑定到的setXXX()方法的参数。
OUT 所提供的SQL语句，它返回一个参数，其值。检索OUT参数值的getXXX()方法。
INOUT 一个参数，同时提供输入和输出值。绑定变量setXXX()方法和检索值getXXX()方法。

下面的代码片段显示了如何利用Connection.prepareCall()方法实例化一个CallableStatement对象，根据上述存储过程： 

CallableStatement cstmt = null;
try {
   String SQL = "{call getEmpName (?, ?)}";
   cstmt = conn.prepareCall (SQL);
   . . .
}
catch (SQLException e) {
   . . .
}
finally {
   . . .
}

String变量SQL的存储过程，参数占位符。

使用CallableStatement对象很像使用PreparedStatement对象。必须执行该语句之前将所有参数值绑定，否则将收到一个SQLException。

如果有IN参数，只要遵循相同的规则和技术，适用于PreparedStatement对象使用setXXX()方法，对应于你要绑定的Java数据类型。

当使用OUT和INOUT参数，必须采用额外CallableStatement对象的方法，registerOutParameter()和registerOutParameter()方法绑定JDBC数据类型，返回存储过程数据类型。

一旦调用存储过程，检索OUT参数值与相应的getXXX()方法。这种方法检索到的SQL类型的值转换到Java的数据类型。

关闭CallableStatement 对象:

正如关闭其他Statement对象，出于同样的原因，也应该关闭CallableStatement对象。

一个简单的调用close()方法将完成这项工作。如果关闭了Connection对象，首先它会关闭CallableStatement对象。然而应该始终明确地关闭CallableStatement对象，以确保适当的清理。 

CallableStatement cstmt = null;
try {
   String SQL = "{call getEmpName (?, ?)}";
   cstmt = conn.prepareCall (SQL);
   . . .
}
catch (SQLException e) {
   . . .
}
finally {
   cstmt.close();
}

JDBC SQL转义语法:

转义语法，使我们可以灵活使用数据库特定的功能不可用，使用标准的JDBC方法和属性。

一般的SQL转义语法格式如下： 

{keyword 'parameters'}

这里之后，你会发现非常有用的，而JDBC编程的转义序列：

d, t, ts 关键字:

他们帮助识别日期，时间和时间戳记文字。正如你所知道的，没有两个DBMS表示时间和日期以同样的方式。此转义语法告诉驱动程序在目标数据库的格式呈现的日期或时间。实例： 

{d 'yyyy-mm-dd'}

其中yyyy=年，mm =月，DD=日。使用语法{D'2009-09-03'}2009年3月9日。
下面是一个简单的例子展示了如何在一个表中插入日期： 


//Create a Statement object
stmt = conn.createStatement();
//Insert data ==> ID, First Name, Last Name, DOB
String sql="INSERT INTO STUDENTS VALUES" +
             "(100,'Zara','Ali', {d '2001-12-16'})";

stmt.executeUpdate(sql);

同样，可以使用以下两种语法，t或ts： 

{t 'hh:mm:ss'}

其中HH=小时，mm =分，ss =秒。使用语法{T'13：30:29'}是下午1点三十分29秒。 

{ts 'yyyy-mm-dd hh:mm:ss'}

这是结合上述两种语法为'd'和't'来表示时间戳的语法。

转义关键字:

该关键字标识LIKE子句中使用的转义字符。使用SQL通配符％，相匹配零个或多个字符时非常有用。例如： 

String sql = "SELECT symbol FROM MathSymbols
              WHERE symbol LIKE '\%' {escape '\'}";
stmt.execute(sql);

如果使用反斜杠字符（\）作为转义字符，也必须使用两个反斜杠字符在Java字符串字面，因为反斜杠也是一个Java转义字符。

fn 关键字:

此关键字代表一个DBMS中使用的标量函数。例如，可以使用SQL函数长度到GE的字符串长度： 

{fn length('Hello World')}

这将返回11，长度字符串“Hello World'。

call 关键字:

关键字是用来调用存储过程。例如，对于需要一个IN参数的存储过程，使用下面的语法： 

{call my_procedure(?)};

需要一个参数并返回一个OUT参数的存储过程，使用下面的语法： 

{? = call my_procedure(?)};

oj 关键字:

使用此关键字来表示外部联接。语法如下： 

{oj outer-join}

外连接表 = table {LEFT|RIGHT|FULL} OUTERJOIN {table | outer-join} 搜索条件。例如： 

String sql = "SELECT Employees 
              FROM {oj ThisTable RIGHT
              OUTER JOIN ThatTable on id = '100'}";
stmt.execute(sql);
