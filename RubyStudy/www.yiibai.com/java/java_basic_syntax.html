Java基本语法

Java的基本语法-当我们考虑一个Java程序，它可以被定义为通过调用彼此的方法，通信的对象的集合

当我们考虑一个Java程序，它可以被定义为通过调用彼此的方法，通信的对象的集合。现在，让我们简要回顾一下类，对象，方法和即时变量意思是什么.

  • 对象 - 对象具有状态和行为。例如：狗有颜色，名称，品种以及行为摇摆，吠叫，吃。对象是类的一个实例.

  • 类- A类可以被定义为一个模板/蓝印描述的行为/状态，其类型支持的对象.

  • 方法- 方法是基本的行为。 A类可以包含许多方法。它是在书面的逻辑方法，数据操纵和执行所有的操作.

  • 即时变量 - 每个对象都有一套独特的即时变数。创建一个对象的状态分配给这些瞬间变量的值.

第一个Java程序:

让我们来看看一个简单的代码，将打印词 Hello World. 

public class MyFirstJavaProgram{

   /* This is my first java program.  
    * This will print 'Hello World' as the output
    * by www.yiibai.com
    */

    public static void main(String []args){
       System.out.println("Hello World"); // prints Hello World
    }
}

让我们看看如何保存文件，编译和运行程序。请按照下面的步骤:

  1. 打开记事本，并添加代码如上.

  2. 文件另存为 : MyFirstJavaProgram.java.

  3. 打开一个命令提示符窗口，并转到您保存类的目录。假设它在 C:\.

  4. 输入 ' javac MyFirstJavaProgram.java ' 和按下进入编译代码。如果在你的代码有没有错误的命令提示符将带你到下一行（假设：路径变量设置）。

  5. 现在输入 ' java MyFirstJavaProgram ' 执行你的程序.

  6. 你将看到 ' Hello World '打印在窗口上. 

C : > javac MyFirstJavaProgram.java
C : > java MyFirstJavaProgram 
Hello World

基础语法:

关于Java程序，它是非常重要的，要牢记以下几点.

  • 大小写敏感 - Java是大小写敏感，这意味着标识Hello，hello在Java中有不同的含义.

  • 类名称 - 对于所有的类名的首字母应该大写. 
    如果几个单词被用来形成一个类的名称，每个内部单词的首字母应该大写.
    例如 class MyFirstJavaClass

  • 方法名称 - 所有方法名应该以小写字母开始. 
    如果几个单词被用来形成方法的名称，然后每个内部单词的首字母应该大写.
    例如 public void myMethodName()

  • 程序文件名 - 程序文件的名称应该完全匹配类名. 
    保存文件时，你应该保存它使用类的名称（请记住Java是大小写敏感）并追加'.java的名称的结尾。 （如果不匹配的文件名和类名，你的程序将不能编译）.
    例如: 假设 'MyFirstJavaProgram' 是一个类名. 那么文件就需要另存为 'MyFirstJavaProgram.java'

  • public static void main(String args[]) -Java程序处理开始从main()方法，这是一个强制每个Java程序的一部分..

Java标识符:

所有的Java组件需要的名字。用于类，变量和方法的名称叫做标识符.

在Java中，有几点要记住标识符。有如下几点:

  • 所有标识符应该开始一个字母（A到Z或a到z），货币字符（$）或下划线（ - ）.

  • 之后的第一个字符的标识符可以有任何字符组合.

  • 一个关键的字不能用作标识符.

  • 标识符是大小写敏感.

  • 合法的标识符的例子:age, $salary, _value, __1_value

  • 非法的标识符的例子 : 123abc, -salary

Java的修饰语:

像其他语言一样，它是可以修改类，方法等使用的修饰符。有两种类别修饰符.

  • 访问修饰符 : defualt, public , protected, private

  • 非访问修饰符 : final, abstract, strictfp

我们将在下一节中寻找到更多细节关于修饰符.

Java枚举:

在Java 5.0中引入枚举。枚举限制一个变量有一个只有几个预定义的值。在此枚举列表的值被称为枚举。
与枚举的使用是有可能在你的代码的错误数量减少.

例如，如果我们考虑鲜果汁店的应用，将有可能限制小型，中型和大型玻璃尺寸。这将确保它不会允许任何人以订购以外的任何大小的小型，中型或大型.

例子: 

class  FreshJuice{
   enum FreshJuiceSize{ SMALL, MEDUIM, LARGE }
   FreshJuiceSize size;
}

/*
 * by www.yiibai.com
 */
public class FreshJuiceTest{
   public static void main(String args[]){
      FreshJuice juice = new FreshJuice();
      juice.size = FreshJuice. FreshJuiceSize.MEDUIM ;
   }
}

注: 作为自己或内部类可以声明枚举。方法，变量，构造，可以定义以及内部枚举。

Java注释

Java支持单行和多行注释非常相似，C和C + +。 Java编译器忽略任何评论内提供的所有字符。 

public class MyFirstJavaProgram{

   /* This is my first java program.
    * This will print 'Hello World' as the output
    * This is an example of multi-line comments.
    */

    public static void main(String []args){
       // This is an example of single line comment
       /* This is also an example of single line comment. */
       /* by www.yiibai.com */
       System.out.println("Hello World"); 
    }
   
}

使用空白行:

行只有空白，可能与注释，被称为一个空白行，和Java完全忽略它.

继承:

在Java中的类可以从类派生。基本上如果你需要创建一个新的类，这里已经是一个类，它有一些你需要的代码，那么它是可以从现有的代码派生新类.

这个概念让你到了一个新的类不必重写代码重用现有类的领域和方法。在这种情况下，现有的类称为超类和派生类被称为子类.

接口:

在Java语言中，一个接口可以被定义为对象之间如何相互沟通的合同。接口起到至关重要的作用，当涉及到继承的概念.

一个接口定义的方法，派生类（子类）应该使用。但实施的方法是完全取决于子类.
