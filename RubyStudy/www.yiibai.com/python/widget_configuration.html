Tkinter小部件配置

Tkinter小部件配置实例代码-要控制一个小部件的外观，您通常使用选项，而不是方法调用。典型的选项包括文本和颜色，大小，命令回调，等来处理选项，所有的核心部件实现了相同的配置界面：

要控制一个小部件的外观，您通常使用选项，而不是方法调用。典型的选项包括文本和颜色，大小，命令回调，等来处理选项，所有的核心部件实现了相同的配置界面：

 配置接口

widgetclass(master, option=value, ...) => widget

这个小工具类创建一个实例，作为一个孩子的主并使用给定的选项。所有的选项都默认值，因此，在最简单的情况下只需要指定主。如果你真的想Tkinter甚至可以离开，使用最近创建的根窗口作为主。需要注意的是name选项可以创建控件时，只能设置。

cget(option) => string

返回一个选项的当前值。选项名称，返回的值，是字符串。为了得到这个名字的选项，使用str（部件）。

configure(option=value, ...), config(option=value, ...)

Set one or more options (given as keyword arguments).

注意某些选项的名称，在Python中的保留字(class, from, ...)。要使用这些关键字参数，只需添加下划线的选项名称（class_，from_，...）。请注意，您不能使用这种方法的名称设置选项，它可以创建控件时，只能设置。

为方便起见，小部件也实现了部分的字典接口。 __setitem__方法映射到配置，而映射到cget的__getitem__。因此，你可以使用下面的语法来设置和查询选项： 

    value = widget[option]
    widget[option] = value

需要注意的是在一个调用Tk的每一个分配结果。如果你想改变多个选项，它通常是一个更好的主意，改变他们与单个调用来config或配置（个人而言，我更喜欢随时更改，时尚的选项）。
下面的字典方法也适用于部件：


keys() => list

返回一个列表的所有选项，可以设置这个小工具。 name选项不包括在此列表中（不能进行查询或修改通过字典接口，无论如何，所以这其实并不重要）。

向后兼容性

在Python1.3中引入的关键字参数。在此之前，股权被传递的窗口部件的构造和配置方法，使用普通的Python字典源代码可能看起来像这样： 

    self.button = Button(frame, {"text": "QUIT", "fg": "red", "command": frame.quit})
    self.button.pack({"side": LEFT})

关键字参数的语法当然是更优雅，而且不容易出错。然而，与现有代码的兼容性，Tkinter仍然支持旧的语法。你不应该使用这种语法在新的方案，即使在某些情况下可能是很有诱惑力的。例如，如果您创建一个自定义的控件，它需要通过配置选项以及它的父类，你可能会喜欢的东西： 

    def __init__(self, master, **kw):
        Canvas.__init__(self, master, kw) # kw is a dictionary

这与当前版本的Tkinter工作就好了，但它的未来版本可能无法正常工作。一个更普遍的做法是使用应用函数： 

    def __init__(self, master, **kw):
        apply(Canvas.__init__, (self, master), kw)

应用函数接受一个函数（一个未绑定的方法，在这种情况下），一个元组参数（其中必须包括自我，因为我们调用一个未绑定的方法），以及可选，提供了关键字参数的字典。
