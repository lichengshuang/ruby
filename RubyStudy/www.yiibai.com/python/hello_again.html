Tkinter又一Hello例子

Tkinter又一Hello例子,当你编写大型程序，它通常是一个好主意，你的代码封装在一个或多个类。

当你编写大型程序，它通常是一个好主意，你的代码封装在一个或多个类。下面的例子是改编自马特·康韦的“Hello World”程序: A Tkinter Life Preserver.

实例3-1. 我们的第二个Tkinter程序 

# File: hello2.py

from Tkinter import *

class App:

    def __init__(self, master):

        frame = Frame(master)
        frame.pack()

        self.button = Button(frame, text="QUIT", fg="red", command=frame.quit)
        self.button.pack(side=LEFT)

        self.hi_there = Button(frame, text="Hello", command=self.say_hi)
        self.hi_there.pack(side=LEFT)

    def say_hi(self):
        print "hi there, everyone - by www.yiibai.com !"

root = Tk()

app = App(root)

root.mainloop()


 执行例子程序

当你运行这个例子时，出现下面的窗口。

图3-1. 运行示例程序 ( Windows7 )

如果你点击右边的按钮，文本“hi there, everyone - by www.yiibai.com！”打印到控制台。如果您按一下左边的按钮，程序停止。

 详细说明

此示例应用程序编写为一类。构造函数（__init__方法）调用父窗口小部件（主），它增加了一些子部件。开始的构造函数创建一个框架构件。帧是一个简单的容器，并在这种情况下，它只用于容纳其他两个小部件。 

    class App:
        def __init__(self, master):

            frame = Frame(master)
            frame.pack()

框架实例存储在一个局部变量框架。创建的小部件后，我们立即调用了包装的方法，使框架可见。
然后，我们创建了两个Button小部件到子框架。 


    self.button = Button(frame, text="QUIT", fg="red", command=frame.quit)
    self.button.pack(side=LEFT)

    self.hi_there = Button(frame, text="Hello", command=self.say_hi)
    self.hi_there.pack(side=LEFT)

这一次，我们通过选择的数目构造作为关键字参数。第一个按钮被标记为“QUIT”，是红的（FG前景的缩写）。第二个被标记为“Hello”。这两个按钮也采取了命令选项。此选项指定一个函数，或（在这种情况下）绑定方法，当单击该按钮时将被调用。
按钮实例存储在实例属性。他们都是加包装(packed)，但这次side= LEFT参数。这意味着他们将被置于作为远离开尽可能在该帧;第一个按钮被放置在帧的左侧边缘和第二个被放置只是第一个右侧（在的左边缘的剩余空间中的帧，这是）。缺省情况下，组装构件，相对于它们的父（这是主帧窗口小部件，和帧本身的按钮）。如果侧边没有给出，它默认为TOP。
“你好”按钮的回调函数给出。它只是简单地打印到控制台每次按下该按钮的消息： 


    def say_hi(self):
        print "hi there, everyone!"

最后，我们提供了一些脚本代码，创建一个Tk根构件，root部件作为其父项的App类的一个实例： 

    root = Tk()

    app = App(root)

    root.mainloop()

最后一次调用是根小部件的mainloop方法。进入的TK事件循环，它的应用程序将继续，直到退出的方法被调用（只需点击“退出”按钮），或关闭窗口。


更多关于小部件引用

在第二个例子中，存储在一个本地变量命名为帧的帧窗口小部件，而按钮的窗口小部件被存储在两个实例的属性。是不是藏在这里有一个严重的问题：__init__函数返回时和框架变量超出范围，会发生什么呢？

别紧张，实际上没有任何需要保持一个参考的部件实例。Tkinter的自动维护一个控件树（通过主动和子女的每一个部件实例的属性），使应用程序的最后一个引用消失时，一个小部件是不会消失的，它必须显式地摧毁了在此之前发生（使用销毁方法）。但是，如果你希望做一些小部件后，已创建，更好地保持自己的小部件的实例参考。

请注意，如果你没有一个部件需要保持一个参考，它可能是诱人的创建和包装上一行： 

    Button(frame, text="Hello", command=self.hello).pack(side=LEFT)

不要将这个操作的结果，你会感到失望，当您尝试使用该值（包方法返回None）。为了安全起见，它可能是更好的包装总是单独建设： 

    w = Button(frame, text="Hello", command=self.hello) 
    w.pack(side=LEFT)


更多关于组件名称

另一种混乱的根源，特别是对那些有一些经验，使用Tcl编程Tk是Tkinter的概念部件的名称。在Tcl中，你必须明确地命名每个插件。例如，下面的Tcl命令创建了一个按钮，命名为“OK”，作为一个孩子，一个小部件命名为“对话框中的”（这又是一个孩子的根窗口，“.”）。 

    button .dialog.ok

相应的Tkinter的调用看起来像这样： 

    ok = Button(dialog)

然而，在Tkinter例子中，“确定”和“对话”窗口小部件实例的名称，而不是实际的窗口小部件的引用。由于Tk本身需要的名字，Tkinter的自动分配一个唯一的名称，每一个新的小部件。另外，在上述的情况下，该对话框名称可能是类似“0.1428748”，并且该按钮可以被命名为“.1428748.1432920”。如果你希望得到一个Tkinter的小部件的全名，只需使用STR功能部件实例： 

    >>> print str(ok)
    .1428748.1432920

（如果打印东西，Python会自动使用STR功能找出打印内容，但很明显操作，如“name= OK”不会做的，所以要确保总是明确地使用str，如果你需要的名称）。

如果你真的需要指定一个部件的名称，你可以用文件名“选项，当您创建的小部件。 （最有可能是唯一的）原因是，如果您需要在Tcl接口编写的代码。

在下面的例子中，产生的小部件被命名为“dialog.ok”（或者，如果你忘了命名为“对话框中，类似”0.1428748“）： 

    ok = Button(dialog, name="ok")

为了避免与Tkinter的命名方案冲突，不使用只包含数字的名称。另外请注意，名称是“创建唯一的”选项，你可以不更改名称，一旦你已经创建了部件。
