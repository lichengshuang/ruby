Ruby方法/函数

Ruby方法是在任何其他的编程语言的函数非常相似。 Ruby方法是使用一个或多个重复的语句捆绑成一个单一的单位。
方法名应该以小写字母开始。如果你开始一个方法的名称以大写字母，Ruby可能会认为它是一个常数，因此可以分析调用不正确。

应界定或声明Ruby方法，然后再调用他们，否则将引发异常未定义的方法调用。

语法： 

def method_name [( [arg [= default]]...[, * arg [, &expr ]])]
   expr..
end
你可以简单地定义一个方法如下： 
def method_name 
   expr..
end
你可以表示一个方法，接受这样的参数： 
def method_name (var1, var2)
   expr..
end

您可以设置默认值，如果没有传递所需的参数调用方法的参数将用于： 

def method_name (var1=value1, var2=value2)
   expr..
end
每当你调用的简单方法，你只写方法的名称如下： 
method_name

然而，当你调用一个带参数的方法，你写的参数，如方法名称： 

method_name 25, 30
使用方法与参数最重要的缺点是，你需要记住参数的数量，每当你调用这些方法。例如，如果一个方法接受三个参数传递只有两个，然后红宝石显示一个错误。

例子： 

#!/usr/bin/ruby

def test(a1="Ruby", a2="Perl")
   puts "The programming language is #{a1}"
   puts "The programming language is #{a2}"
end
test "C", "C++"
test
这将产生以下结果： 
The programming language is C
The programming language is C++
The programming language is Ruby
The programming language is Perl

从方法中返回的值：

 

在Ruby中的每一个方法返回一个默认值。这个返回值将是最后一个语句的值。例如： 

def test
   i = 100
   j = 10
   k = 0
end

这种方法被调用时，将返回的最后声明的变量k。

Ruby return 语句：

Ruby中的return语句用于返回一个或多个值从一个Ruby方法。

语法： 

return [expr[`,' expr...]]

如果有两个以上的表达式，数组中包含这些值将返回值。如果没有表达，零将返回值。

例如： 

return

OR

return 12

OR

return 1,2,3

看看这个代码例子： 

#!/usr/bin/ruby

def test
   i = 100
   j = 200
   k = 300
return i, j, k
end
var = test
puts var

这将输出以下结果： 

100
200
300

可变数目的参数：

假设你声明一个方法，该方法有两个参数。每当你调用这个方法，你需要传递两个参数，与它一起。
然而，Ruby允许你声明的方法与可变数目的参数。让我们来看看本样本： 

#!/usr/bin/ruby

def sample (*test)
   puts "The number of parameters is #{test.length}"
   for i in 0...test.length
      puts "The parameters are #{test[i]}"
   end
end
sample "Zara", "6", "F"
sample "Mac", "36", "M", "MCA"

在这段代码中，你已经宣布接受一个参数的测试方法范例。然而，这个参数是一个可变的参数。这意味着，这个参数可以在任意数量的变量。所以上面的代码将产生以下结果： 

The number of parameters is 3
The parameters are Zara
The parameters are 6
The parameters are F
The number of parameters is 4
The parameters are Mac
The parameters are 36
The parameters are M
The parameters are MCA

类方法：

当类定义之外定义一个方法，该方法默认情况下，标记为私人。另一方面，在类定义中定义的方法公开标示为默认。默认的知名度和私人标记的方法是可以改变的由公共或私人的模块。
每当你要访问一个类的方法，你首先需要实例化的类。然后，使用对象，您可以访问类的任何成员。
红宝石给你一种方法来访问实例化一个类的方法。让我们来看看如何声明和访问一个类的方法： 

class Accounts
   def reading_charge
   end
   def Accounts.return_date
   end
end

看看方法return_date声明。声明一个周期，这是由方法的名称类的名称。你可以访问这个类的方法直接作为如下： 

Accounts.return_date

要访问这个方法，你不必创建对象的类帐户。

Ruby alia语句：

这给方法或全局变量的别名。别名不能被定义在方法体。边缘锯齿的方法，保持目前的方法定义，甚至当方法被重写。禁止编号的全局变量（$ 1，$ 2，...）的别名。覆盖内建的全局变量，可能会导致严重的问题。

语法： 

alias method-name method-name
alias global-variable-name global-variable-name

例子： 

alias foo bar
alias $MATCH $&

在这里，我们已经定义foo别名为bar和$＆$匹配是一个别名。

 

Ruby undef 语句：

这将取消定义的方法。一个undef不能出现在方法体。

undef和别名，接口的类可以独立修改，从超类，但要注意它可能被打破了由自我内部的方法调用程序。


语法： 
undef method-name
例如：
取消定义的方法被称为bar，做到以下几点： 
undef bar
