MySQL管理

MySQL管理实例教程-运行和关闭MySQL服务器,设置MySQL用户帐户等

运行和关闭MySQL服务器：

首先检查MySQL服务器正在运行或不。可以使用下面的命令进行检查： 

 ps -ef | grep mysqld

如果MySQL正在运行，会看到mysqld进程中列出了结果。如果服务器没有运行，那么就可以开始使用下面的命令： 

root@hoat# cd /usr/bin
./safe_mysqld &

现在，如果想关闭一个已经运行的MySQL服务器，那么可以使用以下命令做到这一点： 

root@hoat# cd /usr/bin
./mysqladmin -u root -p shutdown
Enter password: ******

设置MySQL用户帐户:

只需要加入一个新的用户到MySQL数据库MySQL的user表中。

添加新用户guest的密码guest123并给定SQL查询SELECT，INSERT和UPDATE权限，下面是一个例子： 

root@host# mysql -u root -p
Enter password:*******
mysql> use mysql;
Database changed

mysql> INSERT INTO user 
          (host, user, password, 
           select_priv, insert_priv, update_priv) 
           VALUES ('localhost', 'guest', 
           PASSWORD('guest123'), 'Y', 'Y', 'Y');
Query OK, 1 row affected (0.20 sec)

mysql> FLUSH PRIVILEGES;
Query OK, 1 row affected (0.01 sec)

mysql> SELECT host, user, password FROM user WHERE user = 'guest';
+-----------+---------+------------------+
| host      | user    | password         |
+-----------+---------+------------------+
| localhost | guest | 6f8c114b58f2ce9e |
+-----------+---------+------------------+
1 row in set (0.00 sec)

当添加一个新的用户记得要使用PASSWORD()函数由MySQL提供的加密新密码。正如看到在上面的例子中加密的密码mypass6f8c114b58f2ce9e。

请注意FLUSH PRIVILEGES语句。这告诉服务器重新加载授权表。如果不使用它，那么使用新的用户帐户将无法连接到MySQL，至少要等到重新启动服务器，。

以下user表中列的值设置为'Y'执行INSERT查询时，可以更新他们以后使用UPDATE查询，也可以指定其他的权限给新用户。

  • Select_priv
  • Insert_priv
  • Update_priv
  • Delete_priv
  • Create_priv
  • Drop_priv
  • Reload_priv
  • Shutdown_priv
  • Process_priv
  • File_priv
  • Grant_priv
  • References_priv
  • Index_priv
  • Alter_priv

添加用户帐户的另一种方式是通过使用GRANT SQL命令，下面的例子将Ada用户ZARA的与密码zara123为一个特定的数据库称为TUTORIALS。 

root@host# mysql -u root -p password;
Enter password:*******
mysql> use mysql;
Database changed

mysql> GRANT SELECT,INSERT,UPDATE,DELETE,CREATE,DROP
    -> ON TUTORIALS.*
    -> TO 'zara'@'localhost'
    -> IDENTIFIED BY 'zara123';

这也将创建一个项在mysql数据库表中称为用户。

注: MySQL不终止SQL命令，直到给一个分号（;)才结束。

/etc/my.cnf 文件配置:

大多数情况下，不要修改这个文件。默认情况下，将有以下项目： 

[mysqld]
datadir=/var/lib/mysql
socket=/var/lib/mysql/mysql.sock

[mysql.server]
user=mysql
basedir=/var/lib

[safe_mysqld]
err-log=/var/log/mysqld.log
pid-file=/var/run/mysqld/mysqld.pid

在这里，可以指定一个不同的目录中的错误日志，否则你不应该改变任何在该表中。

管理MySQL命令:

这里是重要的MySQL命令列表中，将时常地MySQL数据库管理中使用：

  • USE Databasename : 这将被用来选择一个特定的数据库MySQL中的工作区。

  • SHOW DATABASES: 列出MySQL DBMS中可以访问的数据库。

  • SHOW TABLES: 一旦数据库已选择使用命令，显示数据库中的表。

  • SHOW COLUMNS FROM tablename: 显示的属性，类型的属性，关键信息，是否允许NULL，默认值，表和其他信息。

  • SHOW INDEX FROM tablename: 介绍的细节，包括主键的表上的所有索引。

  • SHOW TABLE STATUS LIKE tablename\G: 在MySQL DBMS的性能和统计的报告中详细说明。
